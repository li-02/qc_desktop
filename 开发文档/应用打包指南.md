## 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| Node.js | >= 18.x | 构建工具运行环境 |
| Python | >= 3.8 | 插补功能依赖 |
| R | >= 4.0 | REddyProc插补功能依赖 |
| electron-builder | 25.x | 打包工具 |

## 打包前准备

### 1. 安装依赖

```bash
# 安装根目录依赖
npm install

# 安装前端依赖
cd frontend && npm install && cd ..
```

### 2. 构建项目

```bash
# 构建前端和后端
npm run build
```

### 3. 检查目录结构

确保以下目录存在且内容完整：

```
qc_desktop/
├── public/
│   ├── dist/          # Electron编译输出
│   └── frontend/dist/ # Vue前端编译输出
├── python/            # Python脚本
│   ├── reddyproc_r_bridge.py
│   ├── timeseries_imputation.py
│   ├── timemixerpp.py
│   └── requirements.txt
└── R-Portable/        # 便携式R环境 (见下文)
```

## R环境准备

REddyProc插补功能需要R环境。打包时会将R环境嵌入安装包。

### 方式一：使用准备脚本（推荐）

```powershell
# 1. 先手动下载R Portable到 R-Portable 目录
# 下载地址: https://sourceforge.net/projects/rportable/

# 2. 运行准备脚本安装REddyProc包
cd scripts
.\prepare_r_portable.ps1
```

### 方式二：手动准备

#### 步骤1：下载R

- **R Portable**: https://sourceforge.net/projects/rportable/
- 或从官网下载: https://cran.r-project.org/bin/windows/base/

#### 步骤2：解压/安装到 `R-Portable` 目录

确保目录结构如下：
```
R-Portable/
├── bin/
│   └── x64/
│       ├── R.dll
│       └── Rscript.exe
├── library/
└── etc/
```

#### 步骤3：安装REddyProc包

```bash
# 使用R-Portable的Rscript安装
R-Portable\bin\x64\Rscript.exe -e "install.packages('REddyProc', repos='https://cloud.r-project.org', dependencies=TRUE)"
```

#### 步骤4：验证安装

```bash
R-Portable\bin\x64\Rscript.exe -e "library(REddyProc); cat('OK\n')"
```

## 打包配置说明

配置文件: `electron-builder.json`

```json
{
  "appId": "com.yourcompany.qc-desktop",
  "productName": "QC Desktop",
  "directories": {
    "output": "release"
  },
  "files": [
    "public/dist/**/*",
    "public/frontend/dist/**/*",
    "node_modules/**/*"
  ],
  "extraResources": [
    {
      "from": "python",
      "to": "python",
      "filter": ["**/*.py", "**/*.pypots", "requirements.txt"]
    },
    {
      "from": "R-Portable",
      "to": "R",
      "filter": ["**/*"]
    }
  ],
  "win": {
    "target": ["nsis"]
  },
  "nsis": {
    "oneClick": false,
    "allowToChangeInstallationDirectory": true
  }
}
```

### 关键配置说明

| 配置项 | 说明 |
|--------|------|
| `extraResources` | 额外资源，会复制到 `resources/` 目录 |
| `python` | Python脚本，打包后位于 `resources/python/` |
| `R-Portable` | R环境，打包后位于 `resources/R/` |
| `nsis.oneClick` | false表示显示安装向导 |

## 常见问题

### Q1: 打包后R环境找不到？

**原因**: R-Portable目录未准备或结构不正确

**解决**: 
1. 确保 `R-Portable/bin/x64/Rscript.exe` 存在
2. 检查electron-builder.json中的extraResources配置

### Q2: REddyProc包加载失败？

**原因**: REddyProc包未安装或依赖缺失

**解决**:
```bash
R-Portable\bin\x64\Rscript.exe -e "install.packages('REddyProc', dependencies=TRUE)"
```

### Q3: 打包体积太大？

**优化方案**:
1. 删除R-Portable中不需要的包
2. 删除R的doc、tests目录
3. 仅保留x64架构

```powershell
# 删除不需要的R组件
Remove-Item -Recurse R-Portable\doc
Remove-Item -Recurse R-Portable\tests
Remove-Item -Recurse R-Portable\bin\i386  # 删除32位
```

### Q4: Python依赖未安装？

应用首次运行时会自动检测并提示安装Python依赖。也可手动安装：

```bash
pip install -r python/requirements.txt
```

## 完整打包流程总结

```bash
# 1. 安装依赖
npm install
cd frontend && npm install && cd ..

# 2. 准备R环境 (首次)
# 下载R Portable到 R-Portable 目录
cd scripts && .\prepare_r_portable.ps1 && cd ..

# 3. 构建项目
npm run build

# 4. 打包
npx electron-builder --win

# 5. 输出位置
# release/QC Desktop Setup 1.0.0.exe
```
