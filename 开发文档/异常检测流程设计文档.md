# 异常检测页面全流程设计文档

> 本文档描述用户从进入异常检测页面到最终展示检测结果的完整交互流程，供前后端开发人员参考。

---

## 一、流程概览

```
用户选择数据集 → 页面初始化 → 加载阈值配置 → 配置/编辑阈值 → 执行检测 → 展示结果 → 查看/删除历史
```

**核心用户路径：**
1. 用户在数据视图中选择一个数据集
2. 切换到「异常检测」面板
3. 系统自动加载该数据集的列阈值配置、检测方法、阈值模板
4. 用户可编辑单列阈值、应用模板、批量清除阈值
5. 用户点击「执行检测」按钮
6. 系统执行阈值检测，返回检测摘要
7. 用户查看检测结果、历史记录

---

#### 1.1 步骤名称与意图

|---------|---------|

**前端代码位置：** `frontend/src/views/data-view/components/panels/OutlierDetectionPanel.vue`

const loadData = async () => {
  if (!datasetInfo.value?.id) return;
  
  await Promise.all([
    outlierStore.loadColumnThresholds(datasetInfo.value.id),
    outlierStore.loadDetectionMethods(),
    outlierStore.loadThresholdTemplates(),
    loadDetectionResults()

### 步骤 5：执行阈值检测 ⭐ 核心流程

#### 5.1 步骤名称与意图
**业务目标：** 基于配置的阈值，对数据集执行异常值检测，识别超出阈值范围的数据点。

#### 5.2 前端行为

| 用户操作 | 前端行为 |
|---------|---------|
| 点击「执行检测」按钮 | 校验是否有配置阈值的列 |
| 校验通过 | 设置 `executing = true`，调用 `executeDetection()` |
| 检测完成 | 更新 `lastDetectionSummary`，刷新检测结果列表 |

**前端代码：**

```typescript
const executeDetection = async () => {
  if (!datasetInfo.value?.id) return;
  
  // 检查是否有配置阈值的列
  const configuredColumns = outlierStore.columnThresholds.filter(
    c => c.min_threshold !== null || c.max_threshold !== null
  );
  
  if (configuredColumns.length === 0) {
    ElMessage.warning("请先为至少一个列配置阈值");
    return;
  }
  
  try {
    executing.value = true;
    const versionId = datasetInfo.value.id; // TODO: 获取实际版本 ID
    
    const result = await outlierStore.executeThresholdDetection(
      String(datasetInfo.value.id),
      String(versionId)
    );
    
    if (result) {
      lastDetectionSummary.value = result.summary;
      await loadDetectionResults();
    }
  } finally {
    executing.value = false;
  }
};
```

#### 5.3 接口定义

| 项目 | 说明 |
|-----|------|
| **Endpoint** | `outlier/execute-threshold-detection` |
| **请求方式** | IPC Invoke |
| **调用方法** | `window.electronAPI.invoke(API_ROUTES.OUTLIER.EXECUTE_THRESHOLD_DETECTION, { datasetId, versionId, columnNames? })` |

**请求参数：**

| 字段名 | 类型 | 必填 | 说明 |
|-------|------|-----|------|
| `datasetId` | `string` | ✅ | 数据集 ID |
| `versionId` | `string` | ✅ | 数据版本 ID |
| `columnNames` | `string[]?` | ❌ | 指定检测的列名（不传则检测所有已配置阈值的列） |

**返回值：**

```typescript
{
  success: boolean;
  data?: {
    resultId: number;           // 检测结果 ID
    summary: {
      totalRows: number;        // 检测的总行数
      columnsChecked: number;   // 检测的列数
      outlierCount: number;     // 发现的异常值总数
      outlierRate: number;      // 异常率 (%)
      columnResults: Array<{
        columnName: string;     // 列名
        outlierCount: number;   // 该列异常值数量
        minThreshold: number | null;
        maxThreshold: number | null;
      }>;
    };
  };
  error?: string;
}
```

#### 5.4 后端处理流程

```
1. 获取列阈值配置
2. 筛选已配置阈值的目标列
3. 创建检测结果记录 (status: RUNNING)
4. 读取数据集 CSV 文件
5. 遍历每列执行阈值检测:
   - 值 < min_threshold → 标记为 BELOW_MIN
   - 值 > max_threshold → 标记为 ABOVE_MAX
6. 批量保存异常值详情 (最多 1000 条)
7. 更新检测结果记录 (status: COMPLETED)
8. 返回检测摘要
```

#### 5.5 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 无配置阈值的列 | 前端拦截，显示警告 |
| 数据集不存在 | 后端返回错误，更新结果状态为 FAILED |
| CSV 文件不存在 | 后端返回错误，更新结果状态为 FAILED |
| 文件读取失败 | 后端返回错误 |

## 四、状态流转图

### 检测结果状态

```
PENDING → RUNNING → COMPLETED
                  ↘ FAILED

COMPLETED → APPLIED → REVERTED
```

| 状态 | 说明 |
|-----|------|
| `PENDING` | 待执行 |
| `RUNNING` | 执行中 |
| `COMPLETED` | 已完成 |
| `FAILED` | 执行失败 |
| `APPLIED` | 已应用（异常值已处理） |
| `REVERTED` | 已撤销 |

## 七、注意事项

1. **阈值验证规则：**
   - `min_threshold` 必须 ≤ `max_threshold`
   - `physical_min` ≤ `warning_min` ≤ `min_threshold`
   - `max_threshold` ≤ `warning_max` ≤ `physical_max`

2. **检测限制：**
   - 异常值详情最多保存 1000 条记录
   - 检测需要至少一个列配置了阈值

3. **性能考虑：**
   - 大文件检测可能耗时较长，前端需显示 loading 状态
   - 历史记录默认只显示最近 5 条
