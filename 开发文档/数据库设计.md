# 数据库设计文档

> **最后更新**：2026-03 | **表数量**：23 张 | **引擎**：SQLite 3 (WAL 模式)

## 1. 概述

本项目采用 **SQLite** 作为本地嵌入式数据库，使用 `better-sqlite3` 作为 Node.js 驱动层。数据库采用**混合存储架构**：

- **SQLite**：存储元数据、配置信息、分析结果、统计摘要
- **文件系统（Parquet）**：存储实际数据文件，数据库仅保存路径引用

数据库管理器为单例模式，代码位于 `electron/core/DatabaseManager.ts`；数据访问层封装在 `electron/repository/` 目录下各 Repository 文件中；TypeScript 类型定义位于 `shared/types/` 目录。

### 1.1 技术栈

| 组件 | 选型 |
|------|------|
| 数据库引擎 | SQLite 3 |
| Node.js 驱动 | better-sqlite3 |
| 日志模式 | WAL (Write-Ahead Logging) |
| 数据文件格式 | Apache Parquet |
| 类型系统 | TypeScript 接口定义 |

### 1.2 表命名规范

| 前缀 | 含义 | 典型表 |
|------|------|--------|
| `sys_` | 系统基础表 | `sys_category`, `sys_dataset`, `sys_settings` |
| `conf_` | 配置表 | `conf_column_setting`, `conf_outlier_detection`, `conf_imputation_method` |
| `biz_` | 业务数据表 | `biz_dataset_version`, `biz_outlier_result`, `biz_workflow` |
| `stat_` | 统计汇总表 | `stat_version_detail` |

### 1.3 通用字段约定（软删除机制）

所有业务表均继承以下基础字段（对应 TypeScript `BaseEntity` 接口）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `created_at` | DATETIME | 记录创建时间，默认 `CURRENT_TIMESTAMP` |
| `updated_at` | DATETIME | 记录最后更新时间，默认 `CURRENT_TIMESTAMP` |
| `deleted_at` | DATETIME | 软删除时间戳，未删除时为 `NULL` |
| `is_del` | BOOLEAN | 软删除标记：`0`=正常，`1`=已删除 |

> **查询约定**：所有正常查询须附加 `WHERE is_del = 0` 条件。

## 3. 数据流与表关系

### 3.1 核心数据流

```
sys_category  (分类)
    └── sys_dataset  (数据集)
            ├── conf_column_setting        (列配置)
            └── biz_dataset_version  ──┐  (数据版本，自引用父版本链)
                    │                  └─ biz_dataset_version (parent)
                    ├── stat_version_detail          (版本统计)
                    ├── biz_outlier_result           (异常检测结果)
                    │       ├── biz_outlier_detail   (单点详情)
                    │       └── biz_outlier_column_stat (列统计)
                    ├── biz_imputation_result        (插补结果)
                    │       ├── biz_imputation_detail (单点详情)
                    │       └── biz_imputation_column_stat (列统计)
                    ├── biz_correlation_result       (相关性分析)
                    └── biz_flux_partitioning_result (通量分割)
```

### 3.2 工作流关系

```
biz_workflow (工作流定义)
    ├── biz_workflow_node (节点定义，有序)
    └── biz_workflow_execution (执行记录，绑定数据集+版本)
            └── biz_workflow_node_execution (节点执行详情，引用版本)
```

### 3.3 配置继承（异常检测）

```
APP（全局） → CATEGORY（分类级） → DATASET（数据集级）
  优先级低         优先级中             优先级高
```

### 4.2 配置模块

#### conf_column_setting — 列配置表

存储数据集每列的质控阈值与元数据，支持三层阈值体系。

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INTEGER | PK AUTOINCREMENT | - | 列配置唯一标识 |
| `dataset_id` | INTEGER | NOT NULL, FK | - | 所属数据集 ID |
| `column_name` | TEXT | NOT NULL | - | 列名 |
| `column_index` | INTEGER | - | NULL | 列在文件中的顺序（从 0 开始） |
| `data_type` | TEXT | - | NULL | 数据类型（`number`/`string`/`datetime`） |
| `min_threshold` | REAL | - | NULL | 基础最小阈值（用户自定义过滤边界） |
| `max_threshold` | REAL | - | NULL | 基础最大阈值 |
| `physical_min` | REAL | - | NULL | 物理最小值（绝对边界，超出即无效） |
| `physical_max` | REAL | - | NULL | 物理最大值 |
| `warning_min` | REAL | - | NULL | 警告最小值（软边界，超出需审核） |
| `warning_max` | REAL | - | NULL | 警告最大值 |
| `unit` | TEXT | - | NULL | 物理单位（如 `°C`、`W/m²`、`μmol/m²/s`） |
| `variable_type` | TEXT | - | NULL | 变量类型（如 `temperature`、`radiation`、`flux`） |
| `is_active` | BOOLEAN | - | 1 | 是否参与计算（`0`=禁用，`1`=启用） |
| `created_at` | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | - | CURRENT_TIMESTAMP | 更新时间 |
| `deleted_at` | DATETIME | - | NULL | 软删除时间 |
| `is_del` | BOOLEAN | - | 0 | 软删除标记 |

**外键**：`dataset_id` → `sys_dataset.id`（级联删除）

**三层阈值体系**：

| 层级 | 字段 | 优先级 | 语义 |
|------|------|--------|------|
| 物理阈值 | `physical_min/max` | 最高 | 硬边界，超出直接视为无效数据 |
| 警告阈值 | `warning_min/max` | 中 | 软边界，超出需人工复核 |
| 基础阈值 | `min/max_threshold` | 低 | 简单过滤边界，用于 UI 配置 |

**TypeScript 接口**：`ColumnSetting`

### 4.3 业务核心模块

#### biz_dataset_version — 数据版本表

记录数据集经过各阶段处理后生成的版本快照，指向文件系统中的 Parquet 文件。版本之间通过 `parent_version_id` 构成有向链。

| 字段 | 类型 | 约束 | 默认值 | 说明 |
|------|------|------|--------|------|
| `id` | INTEGER | PK AUTOINCREMENT | - | 版本唯一标识 |
| `dataset_id` | INTEGER | NOT NULL, FK | - | 所属数据集 ID |
| `parent_version_id` | INTEGER | FK | NULL | 父版本 ID（自引用，追溯处理链） |
| `stage_type` | TEXT | NOT NULL | - | 阶段类型（枚举见下） |
| `file_path` | VARCHAR(256) | NOT NULL | - | Parquet 数据文件路径 |
| `remark` | TEXT | - | NULL | 版本备注 |
| `created_at` | DATETIME | - | CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME | - | CURRENT_TIMESTAMP | 更新时间 |
| `deleted_at` | DATETIME | - | NULL | 软删除时间 |
| `is_del` | BOOLEAN | - | 0 | 软删除标记 |

**外键**：
- `dataset_id` → `sys_dataset.id`（级联删除）
- `parent_version_id` → `biz_dataset_version.id`（自引用）

**`stage_type` 枚举**：

| 值 | 说明 |
|----|------|
| `RAW` | 原始数据（首次导入） |
| `FILTERED` | 异常值过滤后 |
| `QC` | 质量控制（插补等）后 |

## 6. 常用查询示例

### 6.1 查询分类下所有数据集

```sql
SELECT d.* FROM sys_dataset d
WHERE d.category_id = ? AND d.is_del = 0
ORDER BY d.import_time DESC;
```

### 6.2 查询数据集的最新版本

```sql
SELECT v.* FROM biz_dataset_version v
WHERE v.dataset_id = ? AND v.is_del = 0
ORDER BY v.created_at DESC LIMIT 1;
```

### 6.3 获取异常检测有效配置（按优先级解析）

```sql
SELECT * FROM conf_outlier_detection
WHERE is_del = 0 AND is_active = 1
  AND (
    (scope_type = 'APP') OR
    (scope_type = 'CATEGORY' AND scope_id = :categoryId) OR
    (scope_type = 'DATASET' AND scope_id = :datasetId)
  )
ORDER BY
  CASE scope_type WHEN 'DATASET' THEN 3 WHEN 'CATEGORY' THEN 2 ELSE 1 END DESC,
  priority DESC;
```

### 6.4 查询某版本的数据质量统计

```sql
SELECT s.*, v.stage_type FROM stat_version_detail s
JOIN biz_dataset_version v ON s.version_id = v.id
WHERE s.version_id = ? AND s.is_del = 0;
```

### 6.5 查询插补方法及其参数（前端表单渲染）

```sql
SELECT m.*, p.param_key, p.param_name, p.param_type, p.default_value,
       p.min_value, p.max_value, p.options, p.is_advanced
FROM conf_imputation_method m
LEFT JOIN conf_imputation_method_params p ON m.method_id = p.method_id AND p.is_del = 0
WHERE m.is_available = 1 AND m.is_del = 0
ORDER BY m.category, m.priority DESC, p.param_order;
```

### 6.6 查询工作流执行详情（含节点状态）

```sql
SELECT ne.*, n.node_name, n.node_type, n.node_order
FROM biz_workflow_node_execution ne
JOIN biz_workflow_node n ON ne.node_id = n.id
WHERE ne.execution_id = ?
ORDER BY n.node_order ASC;
```

### 6.7 查询数据集的通量分割历史

```sql
SELECT r.* FROM biz_flux_partitioning_result r
WHERE r.dataset_id = ? AND r.is_del = 0
ORDER BY r.executed_at DESC;
```

## 7. 数据库初始化与迁移

### 7.1 初始化流程

数据库管理器 `DatabaseManager`（单例）在应用启动时按以下顺序执行：

1. 在用户数据目录创建 SQLite 文件
2. 设置 `PRAGMA journal_mode = WAL` 提升并发性能
3. 按模块顺序建表：
   - `initCoreTables()` — sys_* 和 biz_dataset_version、stat_version_detail
   - `initSystemSettingsTables()` — sys_settings（含默认值）
   - `initOutlierDetectionTables()` — 异常检测相关 5 张表
   - `initCorrelationAnalysisTables()` — biz_correlation_result
   - `initImputationTables()` — 插补相关 6 张表（含插入默认方法和模型）
   - `initFluxPartitioningTables()` — biz_flux_partitioning_result
   - `initWorkflowTables()` — 工作流 4 张表
4. `migrateSchema()` — 为现有数据库增量添加新字段
