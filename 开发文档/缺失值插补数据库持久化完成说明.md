# 缺失值插补方法数据库持久化完成说明

## 概述

已成功将缺失值插补方法及其参数配置持久化到数据库中，实现了方法和参数的动态管理，避免了硬编码，提高了系统的可扩展性和维护性。

## 完成的数据库表结构

### 1. 新增表：`conf_imputation_method_params`

存储每个插补方法的参数定义信息：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | INTEGER | 主键 |
| `method_id` | TEXT | 所属方法ID |
| `param_key` | TEXT | 参数键名 |
| `param_name` | TEXT | 参数显示名称 |
| `param_type` | TEXT | 参数类型 (number/select/boolean/range) |
| `default_value` | TEXT | 默认值 |
| `min_value` | REAL | 数值类型最小值 |
| `max_value` | REAL | 数值类型最大值 |
| `step_value` | REAL | 数值类型步长 |
| `options` | TEXT | 选择项配置 (JSON) |
| `tooltip` | TEXT | 参数提示信息 |
| `is_required` | INTEGER | 是否必需参数 |
| `is_advanced` | INTEGER | 是否为高级参数 |
| `param_order` | INTEGER | 参数显示顺序 |

### 2. 扩展表：`conf_imputation_method`

新增字段以支持更丰富的方法信息：

- `applicable_data_types`: 适用数据类型 (JSON数组)
- `icon`: 方法图标

## 预设的方法和参数配置

### 支持的方法分类
- **基础方法**: MEAN, MEDIAN, MODE, FORWARD_FILL, BACKWARD_FILL
- **统计方法**: LINEAR, SPLINE, POLYNOMIAL, SEASONAL
- **时序方法**: ARIMA, SARIMA, ETS
- **机器学习**: KNN, RANDOM_FOREST, MICE, MISSFOREST
- **深度学习**: LSTM, GRU, TRANSFORMER, VAE, GAIN

### 参数配置示例

#### ARIMA方法参数
- `autoSelect`: 自动选择参数 (boolean, 默认true)
- `p`: AR阶数 (number, 0-5, 默认1)
- `d`: 差分次数 (number, 0-2, 默认1)
- `q`: MA阶数 (number, 0-5, 默认1)

#### KNN方法参数
- `n_neighbors`: K近邻数量 (number, 1-20, 默认5)

#### 样条插值参数
- `degree`: 样条度数 (select, 1-3, 默认3)

### 2. 为方法添加参数

```sql
INSERT INTO conf_imputation_method_params (
  method_id, param_key, param_name, param_type,
  default_value, is_required, param_order

);
```
