## 1. 概述

### 1.1 执行引擎的职责

执行引擎是 `WorkflowService` 的核心组件，负责：

1. 加载工作流定义和节点列表
2. 创建执行记录和节点执行记录
3. 按 `node_order` 顺序调度节点
4. 管理 `currentVersionId` 在节点间的传递
5. 调用对应的已有 Service 完成节点业务逻辑
6. 记录每个节点的输入/输出版本和执行结果
7. 通过 IPC 事件向前端推送实时进度
8. 处理错误和取消请求

### 1.2 设计约束

- **线性执行**：节点严格按顺序逐个执行，不支持并行
- **失败中断**：任意节点失败则整个工作流中断，后续节点不执行
- **不可回滚**：已完成节点产生的数据版本保留，不自动回滚
- **异步执行**：`execute` 方法立即返回，执行在后台进行

## 2. 执行引擎架构

```text
WorkflowService
│
├── executeWorkflow(workflowId)          ← 入口方法
│   │
│   ├── 1. loadWorkflowWithNodes()       ← 加载定义
│   ├── 2. createExecutionRecords()      ← 创建 DB 记录
│   ├── 3. runPipeline()                 ← 核心调度循环
│   │   │
│   │   ├── FOR EACH enabledNode:
│   │   │   ├── getNodeExecutor(nodeType) ← 获取执行器
│   │   │   ├── executor.execute(ctx)     ← 调用已有 Service
│   │   │   ├── updateNodeExecution()     ← 更新 DB 记录
│   │   │   └── pushProgress()            ← 推送进度
│   │   │
│   │   └── updateExecutionStatus()       ← 更新整体状态
│   │
│   └── 4. finalizeExecution()            ← 收尾处理
│
├── cancelExecution(executionId)          ← 取消执行
│
└── nodeExecutors: Map<NodeType, INodeExecutor>  ← 执行器注册表
    ├── OutlierDetectionExecutor
    ├── ImputationExecutor
    ├── FluxPartitioningExecutor
    ├── CorrelationAnalysisExecutor
    └── ExportExecutor
```

### 3.3 伪代码实现

```typescript
class WorkflowService {
  private nodeExecutors: Map<WorkflowNodeType, INodeExecutor>;
  private cancelFlags: Map<number, boolean>;  // executionId → cancelled

  constructor(
    private workflowRepo: WorkflowRepository,
    private outlierService: OutlierDetectionService,
    private imputationService: ImputationService,
    private fluxService: FluxPartitioningService,
    private correlationService: CorrelationAnalysisService,
    private exportService: ExportService,
  ) {
    this.cancelFlags = new Map();
    this.initNodeExecutors();
  }

  private initNodeExecutors() {
    this.nodeExecutors = new Map();
    this.nodeExecutors.set('OUTLIER_DETECTION',
      new OutlierDetectionExecutor(this.outlierService));
    this.nodeExecutors.set('IMPUTATION',
      new ImputationExecutor(this.imputationService));
    this.nodeExecutors.set('FLUX_PARTITIONING',
      new FluxPartitioningExecutor(this.fluxService));
    this.nodeExecutors.set('CORRELATION_ANALYSIS',
      new CorrelationAnalysisExecutor(this.correlationService));
    this.nodeExecutors.set('EXPORT',
      new ExportExecutor(this.exportService));
  }

  /**
   * 执行工作流（入口方法）
   * 同步创建记录并返回，异步执行流水线
   */
  async executeWorkflow(
    workflowId: number,
    mainWindow: BrowserWindow
  ): Promise<WorkflowExecution> {

    // ===== Phase 1: 准备 =====

    // 1. 加载工作流定义
    const workflow = this.workflowRepo.getWorkflowById(workflowId);
    if (!workflow) throw new Error('WORKFLOW_NOT_FOUND');
    if (workflow.status === 'RUNNING') throw new Error('WORKFLOW_ALREADY_RUNNING');

    // 2. 加载启用的节点
    const allNodes = this.workflowRepo.getNodesByWorkflowId(workflowId);
    const enabledNodes = allNodes.filter(n => n.isEnabled);
    if (enabledNodes.length === 0) throw new Error('WORKFLOW_NO_NODES');
    enabledNodes.sort((a, b) => a.nodeOrder - b.nodeOrder);

    // 3. 创建执行记录
    const executionId = this.workflowRepo.createExecution({
      workflowId,
      status: 'PENDING',
      totalNodes: enabledNodes.length,
      completedNodes: 0,
    });

    // 4. 为每个启用节点创建节点执行记录
    const nodeExecutionIds: number[] = [];
    for (const node of enabledNodes) {
      const neId = this.workflowRepo.createNodeExecution({
        executionId,
        nodeId: node.id,
        status: 'PENDING',
      });
      nodeExecutionIds.push(neId);
    }

    // 5. 更新工作流状态
    this.workflowRepo.updateWorkflowStatus(workflowId, 'RUNNING');

    // 6. 获取执行记录对象（同步返回）
    const execution = this.workflowRepo.getExecutionById(executionId);

    // ===== Phase 2: 异步执行 =====
    this.cancelFlags.set(executionId, false);

    setImmediate(() => {
      this.runPipeline(
        workflow, enabledNodes, executionId,
        nodeExecutionIds, mainWindow
      ).catch(err => {
        console.error('Pipeline unexpected error:', err);
      });
    });

    return execution;
  }

  /**
   * 核心流水线调度循环
   */
  private async runPipeline(
    workflow: Workflow,
    enabledNodes: WorkflowNode[],
    executionId: number,
    nodeExecutionIds: number[],
    mainWindow: BrowserWindow
  ): Promise<void> {

    // 更新执行状态为 RUNNING
    this.workflowRepo.updateExecution(executionId, {
      status: 'RUNNING',
      startedAt: new Date().toISOString(),
    });

    let currentVersionId = workflow.initialVersionId;
    let completedCount = 0;
    let failed = false;

    for (let i = 0; i < enabledNodes.length; i++) {
      const node = enabledNodes[i];
      const nodeExecId = nodeExecutionIds[i];

      // 检查是否被取消
      if (this.cancelFlags.get(executionId)) {
        // 标记当前及后续节点为 SKIPPED
        for (let j = i; j < enabledNodes.length; j++) {
          this.workflowRepo.updateNodeExecution(nodeExecutionIds[j], {
            status: 'SKIPPED',
          });
        }
        this.workflowRepo.updateExecution(executionId, {
          status: 'CANCELLED',
          finishedAt: new Date().toISOString(),
        });
        this.workflowRepo.updateWorkflowStatus(workflow.id, 'FAILED');
        this.pushProgress(mainWindow, {
          executionId,
          currentNodeIndex: i,
          totalNodes: enabledNodes.length,
          nodeStatus: 'SKIPPED',
          message: '执行已取消',
        });
        return;
      }

      // 更新节点执行状态为 RUNNING
      this.workflowRepo.updateNodeExecution(nodeExecId, {
        status: 'RUNNING',
        inputVersionId: currentVersionId,
        startedAt: new Date().toISOString(),
      });

      // 推送进度：节点开始
      this.pushProgress(mainWindow, {
        executionId,
        currentNodeIndex: i,
        totalNodes: enabledNodes.length,
        nodeStatus: 'RUNNING',
        message: `正在执行: ${node.nodeName}`,
      });

      try {
        // 获取节点执行器
        const executor = this.nodeExecutors.get(node.nodeType);
        if (!executor) {
          throw new Error(`未知的节点类型: ${node.nodeType}`);
        }

        // 解析节点配置
        const config = node.configJson ? JSON.parse(node.configJson) : {};

        // 执行节点
        const result = await executor.execute({
          datasetId: workflow.datasetId,
          inputVersionId: currentVersionId,
          config,
          onProgress: (progress, message) => {
            this.pushProgress(mainWindow, {
              executionId,
              currentNodeIndex: i,
              totalNodes: enabledNodes.length,
              nodeStatus: 'RUNNING',
              nodeProgress: progress,
              message,
            });
          },
        });

        // 更新节点执行记录为成功
        this.workflowRepo.updateNodeExecution(nodeExecId, {
          status: 'COMPLETED',
          outputVersionId: result.outputVersionId,
          finishedAt: new Date().toISOString(),
          resultJson: result.resultData
            ? JSON.stringify(result.resultData) : null,
        });

        // 更新 currentVersionId
        if (result.outputVersionId) {
          currentVersionId = result.outputVersionId;
        }

        completedCount++;
        this.workflowRepo.updateExecution(executionId, {
          completedNodes: completedCount,
        });

        // 推送进度：节点完成
        this.pushProgress(mainWindow, {
          executionId,
          currentNodeIndex: i,
          totalNodes: enabledNodes.length,
          nodeStatus: 'COMPLETED',
          message: `完成: ${node.nodeName}`,
        });

      } catch (error: any) {
        // 节点执行失败
        this.workflowRepo.updateNodeExecution(nodeExecId, {
          status: 'FAILED',
          finishedAt: new Date().toISOString(),
          errorMessage: error.message || '未知错误',
        });

        // 推送进度：节点失败
        this.pushProgress(mainWindow, {
          executionId,
          currentNodeIndex: i,
          totalNodes: enabledNodes.length,
          nodeStatus: 'FAILED',
          message: `失败: ${node.nodeName} - ${error.message}`,
        });

        // 标记后续节点为 SKIPPED
        for (let j = i + 1; j < enabledNodes.length; j++) {
          this.workflowRepo.updateNodeExecution(nodeExecutionIds[j], {
            status: 'SKIPPED',
          });
        }

        // 标记执行失败
        this.workflowRepo.updateExecution(executionId, {
          status: 'FAILED',
          finishedAt: new Date().toISOString(),
          errorMessage: `节点 "${node.nodeName}" 执行失败: ${error.message}`,
        });
        this.workflowRepo.updateWorkflowStatus(workflow.id, 'FAILED');

        failed = true;
        break;
      }
    }

    // Phase 3: 收尾 — 全部成功
    if (!failed) {
      this.workflowRepo.updateExecution(executionId, {
        status: 'COMPLETED',
        finishedAt: new Date().toISOString(),
      });
      this.workflowRepo.updateWorkflowStatus(workflow.id, 'COMPLETED');

      this.pushProgress(mainWindow, {
        executionId,
        currentNodeIndex: enabledNodes.length - 1,
        totalNodes: enabledNodes.length,
        nodeStatus: 'COMPLETED',
        message: '工作流执行完成',
      });
    }

    // 清理取消标记
    this.cancelFlags.delete(executionId);
  }

  /**
   * 推送进度事件到前端
   */
  private pushProgress(
    mainWindow: BrowserWindow,
    event: WorkflowProgressEvent
  ) {
    if (mainWindow && !mainWindow.isDestroyed()) {
      mainWindow.webContents.send('workflow:progress', event);
    }
  }

  /**
   * 取消执行
   */
  cancelExecution(executionId: number): boolean {
    const execution = this.workflowRepo.getExecutionById(executionId);
    if (!execution || execution.status !== 'RUNNING') return false;
    this.cancelFlags.set(executionId, true);
    return true;
  }
}
```

## 4. 节点执行器

### 4.1 执行器接口

```typescript
/**
 * 节点执行器标准接口
 * 每种节点类型实现该接口
 */
interface INodeExecutor {
  /** 节点类型标识 */
  readonly nodeType: WorkflowNodeType;

  /**
   * 验证节点配置是否合法
   * @returns 验证结果，包含错误列表
   */
  validateConfig(config: any): {
    valid: boolean;
    errors?: string[];
  };

  /**
   * 执行节点
   * @param context 执行上下文
   * @returns 执行结果（包含输出版本 ID 和结果数据）
   */
  execute(context: NodeExecutionContext): Promise<NodeExecutionResult>;
}

interface NodeExecutionContext {
  /** 数据集 ID */
  datasetId: number;
  /** 输入数据版本 ID */
  inputVersionId: number;
  /** 节点配置（已解析的 JSON 对象） */
  config: any;
  /** 进度回调 */
  onProgress: (progress: number, message: string) => void;
}

interface NodeExecutionResult {
  /** 输出数据版本 ID（数据转换节点有值，分析/输出节点为 null） */
  outputVersionId: number | null;
  /** 执行结果摘要数据 */
  resultData?: Record<string, any>;
}
```

### 4.2 异常检测执行器

```typescript
class OutlierDetectionExecutor implements INodeExecutor {
  readonly nodeType = 'OUTLIER_DETECTION';

  constructor(private outlierService: OutlierDetectionService) {}

  validateConfig(config: OutlierDetectionNodeConfig) {
    const errors: string[] = [];
    if (!config.detectionMethod) {
      errors.push('检测方法不能为空');
    }
    return { valid: errors.length === 0, errors };
  }

  async execute(ctx: NodeExecutionContext): Promise<NodeExecutionResult> {
    const config = ctx.config as OutlierDetectionNodeConfig;

    ctx.onProgress(10, '正在加载数据...');

    // 1. 执行异常检测
    ctx.onProgress(30, '正在检测异常值...');
    const detectionResult = await this.outlierService.executeDetection({
      datasetId: ctx.datasetId,
      versionId: ctx.inputVersionId,
      method: config.detectionMethod,
      targetColumns: config.targetColumns,
      params: config.methodParams,
    });

    // 2. 应用过滤（如果 autoApply = true）
    let outputVersionId: number | null = null;
    if (config.autoApply) {
      ctx.onProgress(70, '正在应用过滤...');
      const applyResult = await this.outlierService.applyFiltering({
        resultId: detectionResult.id,
        datasetId: ctx.datasetId,
        versionId: ctx.inputVersionId,
      });
      outputVersionId = applyResult.newVersionId;
    }

    ctx.onProgress(100, '异常检测完成');

    return {
      outputVersionId,
      resultData: {
        businessResultId: detectionResult.id,
        businessResultTable: 'biz_outlier_result',
        outlierCount: detectionResult.totalOutliers,
        outlierRate: detectionResult.outlierRate,
        applied: config.autoApply,
      },
    };
  }
}
```

### 4.3 缺失值插补执行器

```typescript
class ImputationExecutor implements INodeExecutor {
  readonly nodeType = 'IMPUTATION';

  constructor(private imputationService: ImputationService) {}

  validateConfig(config: ImputationNodeConfig) {
    const errors: string[] = [];
    if (!config.methodId) errors.push('插补方法不能为空');
    return { valid: errors.length === 0, errors };
  }

  async execute(ctx: NodeExecutionContext): Promise<NodeExecutionResult> {
    const config = ctx.config as ImputationNodeConfig;

    ctx.onProgress(10, '正在准备插补...');

    // 1. 执行插补
    const result = await this.imputationService.executeImputation({
      datasetId: ctx.datasetId,
      versionId: ctx.inputVersionId,
      methodId: config.methodId,
      params: config.methodParams,
      targetColumns: config.targetColumns,
    }, (progress, message) => {
      // 映射内部进度到 10-90 范围
      ctx.onProgress(10 + progress * 0.8, message);
    });

    // 2. 应用到数据集（如果 autoApply = true）
    let outputVersionId: number | null = null;
    if (config.autoApply) {
      ctx.onProgress(90, '正在应用插补结果...');
      const applyResult = await this.imputationService.applyResult(result.id);
      outputVersionId = applyResult.newVersionId;
    }

    ctx.onProgress(100, '插补完成');

    return {
      outputVersionId,
      resultData: {
        businessResultId: result.id,
        businessResultTable: 'biz_imputation_result',
        imputedCount: result.imputedCount,
        imputationRate: result.imputationRate,
        applied: config.autoApply,
      },
    };
  }
}
```

class FluxPartitioningExecutor implements INodeExecutor {
  readonly nodeType = 'FLUX_PARTITIONING';

  constructor(private fluxService: FluxPartitioningService) {}

  validateConfig(config: FluxPartitioningNodeConfig) {
    const errors: string[] = [];
    if (!config.methodId) errors.push('分割方法不能为空');
    if (!config.columnMapping?.NEE) errors.push('NEE 列映射不能为空');
    if (!config.columnMapping?.Rg) errors.push('Rg 列映射不能为空');
    if (!config.columnMapping?.Tair) errors.push('Tair 列映射不能为空');
    if (!config.columnMapping?.VPD) errors.push('VPD 列映射不能为空');
    if (!config.siteInfo?.latitude) errors.push('站点纬度不能为空');
    if (!config.siteInfo?.longitude) errors.push('站点经度不能为空');
    return { valid: errors.length === 0, errors };
  }

  async execute(ctx: NodeExecutionContext): Promise<NodeExecutionResult> {
    const config = ctx.config as FluxPartitioningNodeConfig;

    ctx.onProgress(5, '正在准备通量分割...');

    // 调用 FluxPartitioningService
    const result = await this.fluxService.execute({
      datasetId: ctx.datasetId,
      versionId: ctx.inputVersionId,
      method: config.methodId,
      columnMapping: config.columnMapping,
      siteInfo: config.siteInfo,
      options: config.options,
    }, (progress, message) => {
      ctx.onProgress(5 + progress * 0.95, message);
    });

    ctx.onProgress(100, '通量分割完成');

    return {
      outputVersionId: result.newVersionId,
      resultData: {
        businessResultId: result.id,
        businessResultTable: 'biz_flux_partitioning_result',
        method: config.methodId,
      },
    };
  }
}

## 5. 数据版本传递机制

### 5.1 传递规则

```text
┌──────────────────────────────────────────────────────────────┐
│  currentVersionId = workflow.initial_version_id               │
│                                                              │
│  FOR EACH node:                                              │
│    node.input = currentVersionId                             │
│                                                              │
│    IF node.producesVersion:                                  │
│      result = execute(node)                                  │
│      node.output = result.outputVersionId                    │
│      currentVersionId = result.outputVersionId  ← 更新       │
│    ELSE:                                                     │
│      result = execute(node)                                  │
│      node.output = null                                      │
│      currentVersionId = currentVersionId        ← 不变       │
└──────────────────────────────────────────────────────────────┘
```

### 5.2 示例

```text
initial_version_id = 1

Node 1 (异常检测, Transform):
  input = 1, output = 2
  → currentVersionId = 2

Node 2 (相关性分析, Analysis):
  input = 2, output = null
  → currentVersionId = 2 (不变)

Node 3 (缺失值插补, Transform):
  input = 2, output = 3
  → currentVersionId = 3

Node 4 (通量分割, Transform):
  input = 3, output = 4
  → currentVersionId = 4

Node 5 (数据导出, Output):
  input = 4, output = null
  → currentVersionId = 4 (不变)
```

### 5.3 禁用节点

禁用的节点被完全跳过，不参与版本传递链：

```text
Node 1 (启用): input = 1, output = 2 → current = 2
Node 2 (禁用): SKIPPED → current = 2 (不变)
Node 3 (启用): input = 2, output = 3 → current = 3
```

## 6. 进度推送机制

### 6.1 进度事件类型

| 时机 | nodeStatus | message 示例 |
|------|------------|-------------|
| 节点开始执行 | `RUNNING` | `正在执行: 异常检测` |
| 节点内部进度 | `RUNNING` | `正在检测异常值... (40%)` |
| 节点执行完成 | `COMPLETED` | `完成: 异常检测` |
| 节点执行失败 | `FAILED` | `失败: 异常检测 - 数据格式错误` |
| 节点被跳过 | `SKIPPED` | `跳过: 缺失值插补 (已禁用)` |
| 执行被取消 | `SKIPPED` | `执行已取消` |
| 全部完成 | `COMPLETED` | `工作流执行完成` |

### 6.2 前端进度计算

```typescript
// 整体进度百分比计算
const overallProgress = computed(() => {
  if (!executionProgress.value) return 0;
  const { currentNodeIndex, totalNodes, nodeProgress } = executionProgress.value;

  // 已完成节点的比例
  const completedRatio = currentNodeIndex / totalNodes;
  // 当前节点的进度贡献
  const currentRatio = (nodeProgress || 0) / 100 / totalNodes;

  return Math.round((completedRatio + currentRatio) * 100);
});
```

## 7. 错误处理与恢复

### 7.1 错误分类

| 错误类型 | 处理方式 | 示例 |
|----------|----------|------|
| 配置错误 | 执行前校验，阻止执行 | 节点缺少必要参数 |
| 数据错误 | 节点内捕获，标记 FAILED | 列名不存在、数据类型不匹配 |
| Service 错误 | 节点内捕获，标记 FAILED | Python 脚本执行失败 |
| 系统错误 | 外层捕获，标记 FAILED | 数据库连接断开 |

### 7.2 已完成结果保留策略

```text
执行中:
  Node 1: COMPLETED → 产出 v2 ✅ (保留)
  Node 2: COMPLETED → 产出 v3 ✅ (保留)
  Node 3: FAILED    → 无产出 ❌
  Node 4: SKIPPED   → 未执行

用户在 data-view 中可以看到 v2 和 v3，
可以手动在 v3 的基础上继续处理。
```

### 7.3 重新执行

用户修复问题后可以重新执行同一工作流：
1. 会创建新的 `biz_workflow_execution` 记录
2. 从 `initial_version_id` 重新开始
3. 之前执行产生的版本不受影响

## 8. 取消执行机制

### 8.1 取消流程

```text
前端点击[取消]
     │
     ▼
Controller: cancelExecution(executionId)
     │
     ▼
Service: cancelFlags.set(executionId, true)
     │
     ▼
runPipeline 循环中:
  每次循环开始检查 cancelFlags
     │
     ├─ 如果当前节点已经在 Service 调用中:
     │    等待当前节点完成或失败后再取消
     │    (不强制中断底层 Python 脚本)
     │
     └─ 如果当前节点尚未开始:
          直接标记 SKIPPED
```

### 8.2 注意事项

- 取消是**协作式**的，不是强制终止
- 如果某个节点正在执行 Python 脚本（如通量分割），需要等该脚本执行完
- 未来可以增强为强制终止（kill Python 子进程），但第一版不做

## 9. 与现有 Service 的集成细节

### 9.1 依赖注入

在 `ControllerRegistry.ts` 中注入依赖：

```typescript

  this.correlationService,    // 需要新建
  this.exportService,         // 已有
);
this.workflowController = new WorkflowController(
  this.workflowService,
  this.mainWindow
);
```

### 9.2 Service 适配说明

当前部分功能的业务逻辑直接写在 Controller 中（如 `ImputationController`、`FluxPartitioningController`）。工作流执行器需要调用 Service 层方法，因此可能需要：

1. **理想方案**：将 Controller 中的业务逻辑下沉到 Service
2. **临时方案**：在执行器中直接调用 Controller 层的方法（不推荐）
3. **折中方案**：为工作流新建精简版的适配 Service 方法

建议在实施时评估每个 Controller 的业务逻辑复杂度，逐步将逻辑下沉到 Service 层。
