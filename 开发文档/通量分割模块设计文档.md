## 1. 概述

### 1.1 目标

通量分割模块的目标是将涡度协方差 (Eddy Covariance) 系统观测到的**净生态系统碳交换量 (NEE)** 分割为两个组分：

- **总初级生产力 (GPP, Gross Primary Production)** — 植被光合作用固定的碳量
- **生态系统呼吸 (Reco, Ecosystem Respiration)** — 生态系统总呼吸释放的碳量

### 1.2 核心功能

- **两种分割方法**：夜间法 (Reichstein 2005) 和白天法 (Lasslop 2010)
- **自动 Gap-filling**：分割前自动对 NEE、Rg、Tair、VPD 执行 MDS Gap-filling
- **可选 u* 过滤**：通过摩擦速度阈值过滤弱湍流数据
- **列名自动匹配**：智能识别常见通量数据列名变体
- **实时进度反馈**：从 Python/R 脚本到前端的进度推送
- **结果持久化**：分割结果保存为新数据版本并记录到数据库

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue 3 + Element Plus | FluxPartitioningPanel.vue |
| 通信 | Electron IPC | invoke / on 双向通信 |
| 后端 | TypeScript (Electron main) | Controller → Service → Repository |
| 计算 | Python 3 + rpy2 | reddyproc_partitioning.py |
| 核心算法 | R (REddyProc 包) | sMRFluxPartition / sGLFluxPartition |
| 存储 | SQLite | biz_flux_partitioning_result |

## 2. 科学背景

### 2.1 什么是通量分割

涡度协方差系统直接测量的是生态系统与大气之间的**净** CO₂ 交换量 (NEE)。NEE 是两个相反过程的净效应：

- 光合作用（吸收 CO₂）→ 对应 GPP
- 呼吸作用（释放 CO₂）→ 对应 Reco

**通量分割 (Flux Partitioning)** 就是将 NEE 拆分为 GPP 和 Reco 的过程。

### 2.2 为什么需要通量分割

1. **理解碳循环机制**：单看 NEE 无法区分碳吸收与释放的各自贡献
2. **量化光合能力**：GPP 是评估生态系统生产力的关键指标
3. **分析呼吸特征**：Reco 对温度的响应揭示生态系统碳排放规律
4. **模型验证**：地球系统模型和碳循环模型需要独立的 GPP 和 Reco 数据进行验证
5. **生态系统健康评估**：GPP/Reco 比值是评价生态系统碳汇/碳源状态的重要指标

### 2.3 NEE 的物理含义

NEE 遵循符号约定（微气象学符号）：

```
NEE = Reco - GPP
```

- **NEE > 0**：生态系统净释放 CO₂（碳源），多发生在夜间或冬季
- **NEE < 0**：生态系统净吸收 CO₂（碳汇），多发生在生长季白天

因此：

```
GPP = Reco - NEE
Reco = NEE + GPP
```

> **注意**：不同文献和软件中 NEE 的符号约定可能不同。REddyProc 使用上述约定，即 **负 NEE 表示碳吸收**。

## 3. 分割方法原理

### 3.1 夜间法 (Reichstein et al. 2005)

#### 3.1.1 基本思想

夜间没有光合作用（GPP = 0），因此夜间的 NEE 等于 Reco：

```
夜间: NEE_night = Reco_night  (因为 GPP = 0)
```

利用夜间数据建立 **Reco 与温度的关系模型**，然后外推到白天，估算全天候的 Reco，最后由 GPP = Reco - NEE 得到 GPP。

#### 3.1.2 Lloyd-Taylor 呼吸模型

Reco 与温度的关系采用 **Lloyd & Taylor (1994)** 模型：

```
Reco(T) = Rref × exp( E₀ × (1/T_ref - 1/T_obs) )
```

其中：
- `Rref`：参考温度下的呼吸速率 (μmol CO₂ m⁻² s⁻¹)
- `E₀`：温度敏感性参数 (K)，也称为活化能参数
- `T_ref`：参考温度，通常 T_ref - T₀ = 15°C (即 288.15 K - 227.13 K)
- `T_obs`：观测气温减去 T₀ (K)
- `T₀` = -46.02°C (227.13 K)，Lloyd-Taylor 模型的经验常数

#### 3.1.3 拟合过程 (REddyProc sMRFluxPartition)

REddyProc 的 `sMRFluxPartition` 方法执行以下步骤：

1. **筛选夜间数据**：根据站点经纬度和时间计算太阳高度角，筛选太阳在地平线以下的时段
2. **滑动窗口拟合 E₀**：
   - 以连续若干天为一个窗口（默认约 15 天）
   - 在每个窗口内，使用夜间 NEE 和温度数据拟合 Lloyd-Taylor 模型
   - 得到该窗口的 E₀ 值
3. **E₀ 年均值平滑**：
   - 对全年各窗口拟合的 E₀ 取中位数或加权平均，得到一个稳健的年度 E₀ 值
   - 这避免了 E₀ 在数据稀疏时段的不稳定
4. **固定 E₀，重新拟合 Rref**：
   - 使用全年统一的 E₀ 值
   - 在较短的时间窗口内（通常 7 天）重新拟合 Rref
   - 这使 Rref 能够反映呼吸速率的季节变化（如土壤湿度、物候变化等因素）
5. **计算全天候 Reco**：使用拟合得到的参数，对每个时刻计算 Reco(T)
6. **计算 GPP**：GPP = Reco - NEE_filled

#### 3.1.4 输出变量

| 列名 | 含义 |
|------|------|
| `Reco` | 生态系统呼吸估算值 (μmol CO₂ m⁻² s⁻¹) |
| `GPP_f` | 总初级生产力估算值 (μmol CO₂ m⁻² s⁻¹)，已 gap-fill |
| `GPP_fqc` | GPP 质量标记 (0=原始, 1=高质量填充, 2=中等, 3=低质量) |

### 3.2 白天法 (Lasslop et al. 2010)

#### 3.2.1 基本思想

利用白天 NEE 数据同时拟合 **光响应曲线** 和 **呼吸参数**，将光合作用和呼吸在模型层面直接分离。

#### 3.2.2 光响应曲线模型

Lasslop 方法使用修改的 **Michaelis-Menten 矩形双曲线 + VPD 限制**：

```
NEE = -α × β × Rg / (α × Rg + β) + Reco
```

其中引入 VPD 限制后的完整模型为：

```
        ┌  -α × β × Rg / (α × Rg + β) + γ,          VPD ≤ VPD₀
NEE =   │
        └  -α × β₀ × exp(-k(VPD - VPD₀)) × Rg / 
           (α × Rg + β₀ × exp(-k(VPD - VPD₀))) + γ,  VPD > VPD₀
```

参数含义：
- `α`：初始量子产率（光响应曲线的初始斜率），反映低光下的光合效率
- `β`：最大光合速率（光饱和时的光合速率），单位 μmol CO₂ m⁻² s⁻¹
- `γ`：呼吸项（白天基础呼吸），即 Reco 的日间估算
- `Rg`：总辐射 (W m⁻²)
- `VPD₀`：VPD 阈值 (hPa)，超过此值光合开始受 VPD 抑制
- `k`：VPD 敏感系数

#### 3.2.3 VPD 限制机制的物理意义

当空气水汽压差 (VPD) 过高时，植物为避免过度失水会关闭气孔，导致：

1. 蒸腾减少
2. CO₂ 进入叶片的通道受阻
3. 光合速率下降

Lasslop 方法显式地在光响应曲线中考虑了这一效应，使得高 VPD 条件下 GPP 估算更准确。

#### 3.2.4 拟合过程 (REddyProc sGLFluxPartition)

REddyProc 的 `sGLFluxPartition` 方法执行以下步骤：

1. **使用夜间数据估算呼吸参数**：
   - 与夜间法类似，先用夜间数据拟合 Lloyd-Taylor 模型得到 Reco 基线
   - 作为白天模型中呼吸项 γ 的参考
2. **白天数据光响应曲线拟合**：
   - 在滑动时间窗口内，使用白天（Rg > 阈值）的 NEE、Rg、VPD 数据
   - 同时拟合 α、β、γ 和 VPD 限制参数
3. **计算 GPP 和 Reco**：
   - Reco = γ（白天拟合得到的呼吸项）
   - GPP = Reco - NEE

#### 3.2.5 输出变量

| 列名 | 含义 |
|------|------|
| `Reco_DT` | 白天法估算的生态系统呼吸 (μmol CO₂ m⁻² s⁻¹) |
| `GPP_DT_f` | 白天法估算的 GPP (μmol CO₂ m⁻² s⁻¹)，已 gap-fill |
| `GPP_DT_fqc` | GPP 质量标记 |

---

### 3.3 两种方法的对比

| 特性 | 夜间法 (Reichstein) | 白天法 (Lasslop) |
|------|-------------------|-----------------|
| **数据来源** | 夜间 NEE + 温度 | 白天 NEE + 光 + VPD + 温度 |
| **核心模型** | Lloyd-Taylor 呼吸模型 | Michaelis-Menten 光响应 + VPD 限制 |
| **VPD 效应** | 不直接考虑 | 显式建模 |
| **Reco 估算** | 外推夜间关系到白天 | 光响应曲线内部估算 |
| **优势** | 概念清晰，对光响应模型无假设 | 考虑 VPD 对光合的抑制，物理更完备 |
| **劣势** | 夜间数据质量依赖 u* 过滤；假设日夜呼吸关系一致 | 模型参数更多，对数据质量要求更高 |
| **适用场景** | 通用性好，适合大多数站点 | 干旱/半干旱区域更优，VPD 影响显著的站点 |
| **REddyProc 函数** | `sMRFluxPartition()` | `sGLFluxPartition()` |
| **GPP 输出列** | `GPP_f` | `GPP_DT_f` |
| **Reco 输出列** | `Reco` | `Reco_DT` |

#### 使用建议

1. **两种方法都运行**，对比结果，评估不确定性
2. **温带湿润区域**：夜间法通常表现良好
3. **干旱/半干旱区域**：优先考虑白天法（VPD 限制重要）
4. **热带地区**：白天法可能更优（高温高湿导致的 VPD 效应）
5. 通量社区标准产品（如 FLUXNET）通常同时提供两种方法的结果

## 4. 前置条件：MDS Gap-filling

通量分割**必须**在 Gap-filling 之后执行。REddyProc 要求 NEE、Rg、Tair、VPD 四个变量事先经过 MDS (Marginal Distribution Sampling) Gap-filling，生成无缺失的连续时间序列。

本模块在执行分割时**自动完成** Gap-filling，用户无需手动预处理。执行流程：

```
原始数据 (含缺失)
    │
    ▼
sMDSGapFill('NEE')   → NEE_f, NEE_fqc, NEE_fsd
sMDSGapFill('Rg')    → Rg_f, Rg_fqc, Rg_fsd
sMDSGapFill('Tair')  → Tair_f, Tair_fqc, Tair_fsd
sMDSGapFill('VPD')   → VPD_f, VPD_fqc, VPD_fsd
    │
    ▼
sMRFluxPartition() 或 sGLFluxPartition()
    │
    ▼
GPP, Reco 输出
```

MDS Gap-filling 原理参见 Reichstein et al. (2005) 和本项目的「缺失值插补方案设计」文档中的 REddyProc MDS 章节。

## 5. u* 过滤

### 5.1 为什么需要 u* 过滤

夜间弱湍流（低摩擦速度 u*）条件下，涡度协方差系统无法可靠地测量 CO₂ 通量。此时观测的 NEE 往往低估了真实的呼吸量，原因包括：

- 稳定层结导致 CO₂ 在冠层内积累而非向上输送
- 水平平流将 CO₂ 输运到观测范围之外

### 5.2 u* 阈值估算

开启 `ustarFiltering` 选项后，REddyProc 会执行以下步骤：

1. **Bootstrap 采样**：随机抽取 100 次子样本
2. **分段回归**：对每个子样本，使用分段线性回归确定 NEE 不再随 u* 增大而增大的拐点
3. **阈值分布**：得到 u* 阈值的概率分布（5%、50%、95% 分位数）
4. **多场景分割**：分别使用不同 u* 阈值过滤后的数据执行分割

### 5.3 对分割结果的影响

u* 过滤主要影响夜间法的 Reco 估算。过滤掉低 u* 时段的偏低 NEE 后，拟合得到的 Reco 通常会偏高，相应地 GPP 也会增大。多场景分割提供了 GPP/Reco 的不确定性范围。

## 8. API 接口

### 8.1 路由定义

```typescript
FLUX_PARTITIONING: {
  EXECUTE: "fluxPartitioning:execute",
  GET_RESULTS_BY_DATASET: "fluxPartitioning:getResultsByDataset",
  GET_RESULT: "fluxPartitioning:getResult",
  DELETE_RESULT: "fluxPartitioning:deleteResult",
```

### 8.2 执行分割

**路由**: `fluxPartitioning:execute`

**请求体** (`ExecuteFluxPartitioningRequest`):

```typescript
{
  datasetId: number;
  versionId: number;
  methodId: 'NIGHTTIME_REICHSTEIN' | 'DAYTIME_LASSLOP';
  columnMapping: {
    neeCol: string;       // NEE 列名
    rgCol: string;        // 全球辐射列名
    tairCol: string;      // 气温列名
    vpdCol: string;       // VPD 列名
    rhCol?: string;       // 相对湿度列名 (可选，用于计算 VPD)
    ustarCol?: string;    // 摩擦速度列名 (可选)
  };
  siteInfo: {
    latDeg: number;       // 纬度 (°)
    longDeg: number;      // 经度 (°)
    timezoneHour: number; // 时区 (UTC+)
  };
  options?: {
    ustarFiltering?: boolean;  // 是否启用 u* 过滤
  };
}
```

**响应** (`ExecuteFluxPartitioningResponse`):

```typescript
{
  success: boolean;
  resultId?: number;    // 结果记录 ID
  message?: string;     // 成功消息
  error?: string;       // 错误消息
}
```

### 8.3 进度事件

**事件名**: `fluxPartitioning:progress`

**事件数据** (`FluxPartitioningProgressEvent`):

```typescript
{
  resultId: number;
  stage: 'checking' | 'loading' | 'preparing' | 'gapfilling' | 'partitioning' | 'saving' | 'completed';
  progress: number;   // 0-100
  message: string;    // 中文进度描述
}
```

### 8.4 查询结果列表

**路由**: `fluxPartitioning:getResultsByDataset`

**请求**: `{ datasetId: number; limit?: number; offset?: number }`

**响应**: `{ success: boolean; data?: FluxPartitioningResult[]; error?: string }`

### 8.5 查询单个结果

**路由**: `fluxPartitioning:getResult`

**请求**: `resultId: number`

**响应**: `{ success: boolean; data?: FluxPartitioningResult; error?: string }`

### 8.6 删除结果

**路由**: `fluxPartitioning:deleteResult`

**请求**: `resultId: number`

**响应**: `{ success: boolean; error?: string }`

## 10. 输入输出规范

### 10.1 输入变量要求

| 变量 | 是否必需 | 单位 | 说明 |
|------|---------|------|------|
| **NEE** | 必需 | μmol CO₂ m⁻² s⁻¹ | 净生态系统碳交换量 |
| **Rg** | 必需 | W m⁻² | 总辐射 (短波入射辐射) |
| **Tair** | 必需 | °C | 气温 |
| **VPD** | 必需 | hPa | 水汽压差。若无 VPD 但有 RH，系统自动计算 |
| RH | 可选 | % | 相对湿度 (用于计算 VPD) |
| Ustar | 可选 | m s⁻¹ | 摩擦速度 (u* 过滤时需要) |
| 时间列 | 必需 | - | 可被 pandas 解析的时间格式 |

### 10.2 输出变量说明

#### 夜间法输出

| 输出列 | 含义 | 单位 |
|--------|------|------|
| `NEE_f` | Gap-filled NEE | μmol CO₂ m⁻² s⁻¹ |
| `NEE_fqc` | NEE 填充质量标记 | 0-3 |
| `Rg_f` | Gap-filled 辐射 | W m⁻² |
| `Tair_f` | Gap-filled 气温 | °C |
| `VPD_f` | Gap-filled VPD | hPa |
| `Reco` | 生态系统呼吸 (夜间法) | μmol CO₂ m⁻² s⁻¹ |
| `GPP_f` | 总初级生产力 (夜间法) | μmol CO₂ m⁻² s⁻¹ |
| `GPP_fqc` | GPP 质量标记 | 0-3 |

#### 白天法输出

| 输出列 | 含义 | 单位 |
|--------|------|------|
| `NEE_f` | Gap-filled NEE | μmol CO₂ m⁻² s⁻¹ |
| `Reco_DT` | 生态系统呼吸 (白天法) | μmol CO₂ m⁻² s⁻¹ |
| `GPP_DT_f` | 总初级生产力 (白天法) | μmol CO₂ m⁻² s⁻¹ |
| `GPP_DT_fqc` | GPP 质量标记 | 0-3 |

#### 质量标记说明

| 值 | 含义 |
|----|------|
| 0 | 原始观测值（未被填充） |
| 1 | 高质量填充（良好的气象条件匹配） |
| 2 | 中等质量填充 |
| 3 | 低质量填充（仅基于均值/中位数等粗略方法） |
