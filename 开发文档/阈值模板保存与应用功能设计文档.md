## 1. 功能概述

### 1.1 背景

当前静态阈值过滤的模板功能仅支持两个硬编码的内置模板（"标准模板" 和 "严格模板"），用户无法保存自己配置好的阈值方案供后续复用。当用户在不同数据集上需要使用相同的阈值配置时，必须逐列手动重新设置，效率低下。

### 1.2 功能目标

| 功能 | 说明 |
|------|------|
| **保存为模板** | 将当前数据集中已配置阈值的列导出为一个可复用的模板 |
| **模板列表管理** | 查看、删除用户自定义模板 |
| **一键应用模板** | 选择模板后自动按列名匹配，将模板中的阈值写入目标数据集 |
| **智能匹配** | 按 `column_name` 精确匹配，模板中没有的指标跳过，不覆盖未匹配列 |

### 1.3 与现有模板的关系

- **内置模板**（standard / strict）：保持不变，继续作为通量数据常用预设
- **用户自定义模板**：新增功能，存储在数据库中，用户可自由创建和删除
- **UI 展示**：模板下拉菜单中分组显示「内置模板」和「我的模板」

## 3. 数据库设计

### 3.1 新增表：`conf_threshold_template`

```sql
CREATE TABLE IF NOT EXISTS conf_threshold_template (
  id                INTEGER PRIMARY KEY AUTOINCREMENT,  -- 模板唯一标识
  name              TEXT NOT NULL,                      -- 模板名称
  description       TEXT,                               -- 模板描述（可选）
  template_data     TEXT NOT NULL,                      -- 模板数据 (JSON格式)
  source_dataset_id INTEGER,                            -- 来源数据集ID（仅记录来源，可为NULL）
  created_at        DATETIME DEFAULT CURRENT_TIMESTAMP, -- 创建时间
  updated_at        DATETIME DEFAULT CURRENT_TIMESTAMP, -- 更新时间
  deleted_at        DATETIME,                           -- 删除时间
  is_del            BOOLEAN DEFAULT 0                   -- 软删除标记
);
```

### 3.2 template_data JSON 格式

```json
{
  "Ta": { "min": -40, "max": 50, "unit": "°C" },
  "RH": { "min": 0, "max": 100, "unit": "%" },
  "SW_IN": { "min": 0, "max": 1400, "unit": "W/m²" },
  "NEE": { "min": -40, "max": 30, "unit": "μmol/m²/s" }
}
```

每个 key 为列名（`column_name`），value 包含：
- `min`: 最小阈值（对应 `min_threshold`）
- `max`: 最大阈值（对应 `max_threshold`）
- `unit`: 单位（可选）

### 3.3 索引

```sql
CREATE INDEX IF NOT EXISTS idx_threshold_template_name
  ON conf_threshold_template(name);
```

## 5. 完整操作流程

### 5.1 保存模板流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  配置好阈值  │ ──▶ │ 点击"保存   │ ──▶ │ 输入模板    │ ──▶ │  保存成功   │
│  (各列阈值)  │     │  为模板"    │     │ 名称/描述   │     │  提示反馈   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

**详细步骤**：
1. 用户在静态阈值配置表格中为各列设置好 min/max 阈值
2. 点击"保存为模板"按钮
3. 弹出对话框，输入模板名称（必填）和描述（可选）
4. 前端调用 `SAVE_AS_TEMPLATE` API
5. 后端读取当前数据集中所有已配置阈值（`min_threshold` 或 `max_threshold` 不为 null）的列
6. 序列化为 JSON 存入 `conf_threshold_template` 表
7. 前端刷新模板列表，显示成功提示

### 5.2 应用模板流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  打开新数据  │ ──▶ │ 打开模板    │ ──▶ │ 选择模板    │ ──▶ │  确认应用   │
│  集的阈值页  │     │ 下拉菜单    │     │ (内置/自定义)│     │  匹配写入   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

**详细步骤**：
1. 用户切换到另一个数据集的阈值配置页面
2. 点击模板下拉菜单
3. 选择一个用户自定义模板（或内置模板）
4. 弹出确认框，显示将要匹配的列数信息
5. 前端调用 `APPLY_USER_TEMPLATE` API
6. 后端遍历目标数据集的列配置，按 `column_name` 匹配模板中的 key
7. 匹配到的列更新 `min_threshold`、`max_threshold`、`unit`
8. 返回匹配成功数和跳过数
9. 前端刷新阈值表格，显示结果提示

## 6. API 详细说明

### 6.1 保存为模板

**路由**: `outlier/save-as-template`

**请求参数**:

```typescript
{
  datasetId: string;   // 数据集 ID（读取该数据集的列阈值配置）
  name: string;        // 模板名称（必填）
  description?: string; // 模板描述（可选）
}
```

**返回值**:

```typescript
{
  success: true,
  data: {
    id: number;          // 新创建的模板ID
    columnCount: number; // 保存了多少列的配置
  }
}
```

**业务逻辑**:
1. 验证 `name` 不为空
2. 读取 `conf_column_setting` 中 `dataset_id` 匹配且 `min_threshold` 或 `max_threshold` 不为 null 的行
3. 如果没有任何已配置的列，返回错误提示
4. 将各列的 `column_name` → `{ min, max, unit }` 组装为 JSON
5. 插入 `conf_threshold_template` 表
6. 返回模板 ID 和列数

### 6.2 获取用户模板列表

**路由**: `outlier/get-user-templates`

**请求参数**: 无

**返回值**:

```typescript
{
  success: true,
  data: {
    templates: Array<{
      id: number;
      name: string;
      description: string | null;
      columnCount: number;       // 模板包含的列数
      columnNames: string[];     // 模板包含的列名列表（用于预览）
      sourceDatasetId: number | null;
      createdAt: string;
    }>
  }
}
```

**业务逻辑**:
1. 查询 `conf_threshold_template` 中 `is_del = 0` 的记录
2. 解析每条记录的 `template_data` JSON，计算列数并提取列名
3. 按 `created_at DESC` 排序返回

### 6.3 删除用户模板

**路由**: `outlier/delete-user-template`

**请求参数**:

```typescript
{
  templateId: string;  // 模板 ID
}
```

**返回值**:

```typescript
{
  success: true,
  data: { success: true }
}
```

**业务逻辑**:
1. 验证模板存在
2. 软删除：`is_del = 1, deleted_at = CURRENT_TIMESTAMP`

### 6.4 应用用户模板

**路由**: `outlier/apply-user-template`

**请求参数**:

```typescript
{
  datasetId: string;   // 目标数据集 ID
  templateId: string;  // 模板 ID
}
```

**返回值**:

```typescript
{
  success: true,
  data: {
    appliedCount: number;  // 成功匹配并更新的列数
    skippedCount: number;  // 模板中有但数据集中不存在的列数
    totalInTemplate: number; // 模板中的总列数
  }
}
```

**业务逻辑**:
1. 从 `conf_threshold_template` 读取模板数据
2. 解析 `template_data` JSON
3. 读取目标数据集的 `conf_column_setting`
4. 遍历模板中的每个 key（列名）:
   - 在目标数据集中按 `column_name` 精确匹配
   - 匹配到 → 更新 `min_threshold`, `max_threshold`, `unit`
   - 未匹配到 → 跳过，计入 skippedCount
5. 批量更新数据库
6. 返回统计结果

## 7. 类型定义

/**
 * 用户自定义阈值模板表 (conf_threshold_template)
 */
export interface ThresholdTemplateRecord extends BaseEntity {
  id: number;
  name: string;
  description?: string;
  template_data: string;           // JSON 格式
  source_dataset_id?: number;
  created_at: string;
}

/**
 * 模板数据条目（单列）
 */
export interface ThresholdTemplateEntry {
  min: number;
  max: number;
  unit?: string;

/**
 * 模板数据（解析后的 template_data）

export type ThresholdTemplateData = Record<string, ThresholdTemplateEntry>;

 * 用户模板列表项（前端展示用）

export interface UserTemplateListItem {
  id: number;
  name: string;
  description: string | null;
  columnCount: number;
  columnNames: string[];
  sourceDatasetId: number | null;
  createdAt: string;
}

## 9. 匹配策略

### 9.1 匹配规则

| 优先级 | 匹配方式 | 说明 |
|--------|----------|------|
| 1 | `column_name` 精确匹配 | 模板 key 与数据集列名完全相同 |

### 9.2 匹配行为

- **匹配到**：更新该列的 `min_threshold`、`max_threshold`、`unit`
- **未匹配到**：跳过，不影响目标数据集中该列的现有配置
- **目标列已有配置**：覆盖为模板中的值（应用前有确认弹窗提醒）

### 9.3 示例

**模板内容**:
```json
{ "Ta": { "min": -40, "max": 50, "unit": "°C" }, "RH": { "min": 0, "max": 100, "unit": "%" }, "CO2": { "min": 350, "max": 550, "unit": "ppm" } }
```

**目标数据集列**: `Ta`, `RH`, `SW_IN`, `PPFD`

**匹配结果**:
- `Ta` ✅ 匹配，更新阈值
- `RH` ✅ 匹配，更新阈值
- `CO2` ⏭ 跳过（数据集中不存在该列）
- `SW_IN` — 不受影响（模板中没有该列）
- `PPFD` — 不受影响（模板中没有该列）

**返回**: `{ appliedCount: 2, skippedCount: 1, totalInTemplate: 3 }`
